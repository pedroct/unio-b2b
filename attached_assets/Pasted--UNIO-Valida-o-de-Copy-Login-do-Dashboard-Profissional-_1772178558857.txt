# UNIO — Validação de Copy: Login do Dashboard Profissional

**Plataforma:** Web (Dashboard Profissional)  
**Tela:** Login / Autenticação  
**Data:** 27/02/2026  
**Escopo:** Textos, copy, UX writing e ajustes visuais menores — não alterar lógica, chamadas de API ou estilos estruturais.

---

## Resumo da análise

A tela de login do dashboard profissional está funcional e visualmente alinhada à identidade UNIO (paleta verde/terra, tipografia limpa). No entanto, foram identificados **9 ajustes** de copy, UX writing e consistência com as diretrizes da marca.

## Funcionalidades ausentes a considerar

### A. Link "Esqueci minha senha"

A tela atual **não possui** link de recuperação de senha/acesso. Segundo as diretrizes de UX do UNIO (seção 3.8 — Mensagens de Erro), o fluxo de login deve oferecer alternativa em caso de falha.

**Sugestão:** Adicionar abaixo do helper text:
```
Não consegue acessar? Entre em contato com o suporte.
```
Ou, se houver fluxo de recuperação:
```
Esqueceu seu CPF? Recuperar acesso
```

### B. Mensagens de erro inline

Implementar validações inline conforme padrão UNIO `[O que aconteceu] + [O que fazer agora]`:

| Situação | Mensagem de erro |
|----------|------------------|
| Registro inválido | `Registro profissional não encontrado. Verifique o número e tente novamente.` |
| UF não selecionada | `Selecione o estado do seu registro.` |
| CPF inválido | `CPF inválido. Verifique os números e tente novamente.` |
| Credenciais incorretas | `Registro ou CPF incorretos. Verifique seus dados e tente novamente.` |
| Sem conexão | `Sem conexão. Verifique sua internet e tente de novo.` |

### C. Estado de loading do botão

Ao clicar "Entrar", o botão deve:
- Exibir spinner ou texto `Entrando...`
- Ficar desabilitado durante a requisição
- Não usar "Carregando" ou "Aguarde"

## Ajustes visuais menores (observados)

### D. Ícone no campo de senha

O campo de senha possui dois ícones à direita: um ícone de fingerprint/biometria (🛡️) e um ícone de visibilidade (👁️). 

**Recomendações:**
- O ícone de biometria não faz sentido em login web — remover, a menos que haja integração com WebAuthn/passkeys
- Manter apenas o ícone de visibilidade (olho) para toggle show/hide da senha

### E. Contraste do placeholder

Verificar se o placeholder `CRM-12345` tem contraste suficiente com o fundo do input (mínimo WCAG AA 4.5:1 para texto, ou 3:1 para texto grande).

## Regras gerais aplicadas (referência)

- Títulos: sem ponto final
- Labels: sentence case (apenas primeira palavra em maiúscula)
- Botões: sentence case, verbo no infinitivo
- Nenhum texto em ALL CAPS
- Helper texts: frases completas com ponto final
- Placeholders: descritivos, com prefixo "Ex:" quando aplicável
- Mensagens de erro: `[O que aconteceu] + [O que fazer agora]`
- Tom: acolhedor, profissional, sem jargão técnico desnecessário
