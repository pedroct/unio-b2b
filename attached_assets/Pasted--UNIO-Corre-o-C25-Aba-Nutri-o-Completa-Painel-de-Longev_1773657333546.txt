# UNIO — Correção C25: Aba Nutrição Completa
## Painel de Longevidade — Auditoria e Refinamento

---

**Prioridade:** Média-Alta
**Tela:** Painel de Longevidade → Aba Nutrição (B2B web)
**Contexto:** A aba Nutrição é a mais completa e visualmente rica do Painel, com KPIs, gráfico de evolução, correlações cruzadas e tabela de registros. O conteúdo está excelente — os ajustes são de refinamento para alinhar com os padrões UNIO já definidos nas outras abas.

---

## Análise Geral

### O que está muito bom
```
✅ Conteúdo clínico denso e relevante (proteína relativa, aderência, macros, correlações)
✅ Gráfico "Proteína Relativa — Evolução 30d" com faixas de referência coloridas
✅ Seção "Correlações" com 3 gráficos cruzados — excelente para análise do profissional
✅ Gráfico "Proteína Relativa ↔ Massa Magra" com eixo duplo — visão avançada
✅ Gráfico "Carboidrato ↔ Glicemia" conectando nutrição e biometria
✅ Tabela "Linha do tempo de registros" com aderência e status por dia
✅ Donut de distribuição de macros (Proteína 24%, Carbo 49%, Gordura 27%)
✅ Filtro de período (7d, 30d, 90d) no topo
✅ Badge "Elevado" na proteína relativa (classificação funcional)
✅ Barra de progresso na aderência calórica (101%)
✅ Chevron de expansão na tabela (>) para detalhe por dia
```

### Desvios a corrigir
```
🔴 Labels de seção ausentes — "Correlações" usa ícone+bold em vez do padrão
🔴 Cores dos gráficos: verdes genéricos em vez de tokens UNIO
🔴 Badge "Elevado" em amarelo/verde — verificar se é o token correto
🔴 Status na tabela ("Baixa", "Alta", "Moderada") em vermelho/verde — cores fora do padrão
🔴 Cards de KPI sem accent bar do módulo
⚠️ Tipografia dos gráficos parece system font
⚠️ Legenda do gráfico de evolução com cores que não correspondem aos tokens
⚠️ Donut de macros com cores que podem não ser os tokens exatos
⚠️ Título "Linha do Tempo de Registros" em Title Case
⚠️ "Período:" como label antes dos filtros — desnecessário
```

## Bloco 2 — Gráfico "Proteína Relativa — Evolução 30d"

### Correções

**C25.6 — Título do gráfico:**
```
ANTES:  "📈 Proteína Relativa — Evolução 30d ⓘ"
DEPOIS: "Proteína relativa — evolução 30 dias ⓘ"
        Inter SemiBold 16px, #2F5641
        Sem emoji 📈 — usar sentence case
        "30d" → "30 dias" por extenso
```

**C25.7 — Faixas de referência:**

As faixas coloridas no fundo do gráfico estão corretas no conceito. Alinhar cores com tokens:

```
< 1.2 g/kg (Insuficiente):  fill #D97952 com 6% opacidade
1.2–1.6 g/kg (Subótimo):    fill #D9A441 com 6% opacidade
1.6–2.2 g/kg (Adequado):    fill #4CA785 com 6% opacidade
> 2.2 g/kg (Elevado):       fill #D9A441 com 6% opacidade
```

**C25.8 — Linha do gráfico:**
```
Cor da linha:   #648D4A (primaryLight — cor da nutrição)
Stroke:         2px
Pontos:         Circle 5px, cor varia com a zona do valor
                Na zona adequada: #4CA785
                Na zona subótima: #D9A441
                Na zona insuficiente: #D97952

  1.6 g/kg (limite inferior adequado): dashed #4CA785 50% opacidade
  2.2 g/kg (limite superior adequado): dashed #D9A441 50% opacidade

**C25.9 — Legenda do gráfico:**

  ▪ #D97952  "< 1.2 g/kg — Insuficiente"
  ▪ #D9A441  "1.2–1.6 g/kg — Subótimo"

  ▪ #D9A441  "> 2.2 g/kg — Elevado"

**C25.10 — Eixos do gráfico:**
```

## Padrões de Cores para Gráficos Nutricionais

Referência rápida para a LLM do frontend — usar exclusivamente estas cores em gráficos da aba Nutrição:

```
Proteína (linha/barra):    #648D4A (primaryLight / modNutricao)
Carboidrato (linha/barra): #AD8C48 (secondary / dourado)
Gordura (linha/barra):     #D97952 (accent / terracota)

Estas são as mesmas cores dos macro badges no app mobile:
  PROT = verde (#648D4A)
  CARB = dourado (#AD8C48)
  GORD = terracota (#D97952)

Massa magra (correlação):  #3D7A8C (modCorpo)
Glicemia (correlação):     Cor da zona (success/warning/accent)

Faixas de referência:      Cor da zona com 6–8% opacidade
Linhas de referência:      Cor da zona com 50% opacidade, dashed
Grid do gráfico:           #E8EBE5, 0.5px, dashed
Eixos:                     Inter Regular 11px, #8B9286
```
