# Documentação de Integração - Módulo Clientes (Listagem Frontend WEB)

Esta documentação define o contrato do endpoint responsável por fornecer a listagem geral de clientes (pacientes) vinculados a um profissional de saúde, alimentando a tabela principal do Dashboard (menu "Clientes").

---

## 1. Visão Geral (Listagem de Clientes)
O endpoint retorna um *array* contendo o resumo dos dados cadastrais, indicadores de engajamento preenchidos e o status da conta de cada paciente sob a tutela do profissional autenticado.

*   **Método:** `GET`
*   **Acesso:** Protegido (Requer token JWT Bearer do Profissional)
*   **URL Base:** `/api/profissional/clientes`

### 1.1 Exemplo de Resposta de Sucesso (200 OK)
A resposta é um `Array` de objetos.

## 3. Comportamento e Tratamento de Erro (HTTP)

*   **`401 Unauthorized`**: O Token JWT não foi enviado pelo Axios/Fetch na chave `Authorization: Bearer <token>` ou expirou. O frontend deve forçar o fluxo de Refresh ou Logout do profissional.
*   **`403 Forbidden`**: Tentativa de burlar privilégios. O endpoint é travado apenas para usuários contendo o Perfil `"medico"`, `"personal"`, ou `"nutricionista"`.
*   **Filtros de Tabela**: O Backend não preverá *Query Strings* (ex: `?search=Ana` ou `?sort=name`) nesta entrega inicial. A busca, filtros de tags e ordenações (A-Z) devem ser performadas **Client-Side** (React/Vue/Angular) na listagem renderizada após a captura total do Array `200 OK`.
