Todas as correções de copy (placeholders, labels, mensagens) são da Fase 1 e podem ser implementadas agora.

## Parte 2 — Funções utilitárias (Fase 1)

Criar 3 funções em arquivo compartilhado (ex: `utils/formatters.ts`). Essas funções serão usadas em **todas as telas** que exibem dados nutricionais: este modal, Prescrição Alimentar, e telas futuras.

### 2.1 `formatFoodName(descricao: string): string`

Formata o nome bruto do alimento para exibição legível. Usada como fallback quando o bloco `apresentacao` não existir.

```typescript
export const formatFoodName = (descricao: string): string => {
  if (!descricao) return '';

  return descricao
    // Expandir abreviações
    .replace(/\bc\/\s*/g, 'com ')
    .replace(/\bs\/\s*/g, 'sem ')
    // Remover origem geográfica no final
    .replace(/,\s*Brasil\s*$/i, '')
    // Remover metadados técnicos
    .replace(/,\s*todas as variedades/gi, '')
    // Limpar espaços duplos
    .replace(/\s{2,}/g, ' ')
    .trim();
};
```

**Importante:** Esta função NÃO remove parênteses nem separa modo de preparo — isso será responsabilidade do backend via bloco `apresentacao`. A função apenas resolve os problemas mais críticos de legibilidade (abreviações e metadados inúteis).

---

### 2.2 `formatNutrient(value: number): string`

Formata valores nutricionais para exibição em pt-BR.

```typescript
export const formatNutrient = (value: number): string => {
  if (value == null || isNaN(value)) return '0';
  if (value === 0) return '0';
  if (Number.isInteger(value)) return value.toString();

  // Uma casa decimal, vírgula como separador
  const formatted = value.toFixed(1);
  // Remover .0 desnecessário
  const cleaned = formatted.replace(/\.0$/, '');
  // Trocar ponto por vírgula
  return cleaned.replace('.', ',');
};
```

**Regras:**

| Entrada | Saída | Regra |
|---------|-------|-------|
| `279.00` | `279` | Inteiro — sem decimal |
| `279` | `279` | Inteiro — sem decimal |
| `31.20` | `31,2` | Fracionário — 1 casa, vírgula |
| `0.00` | `0` | Zero — sem decimal |
| `0` | `0` | Zero — sem decimal |
| `5.05` | `5,1` | Arredondamento para 1 casa |
| `null` | `0` | Fallback seguro |

---

### 2.3 `formatUnit(unit: string): string`

Força todas as unidades de medida para minúsculas.

```typescript
export const formatUnit = (unit: string): string => {
  if (!unit) return '';
  return unit.toLowerCase();
};
```

**Exemplos:**

| Entrada | Saída |
|---------|-------|
| `Colher De Sopa` | `colher de sopa` |
| `Copo De Requeijão` | `copo de requeijão` |
| `Xícara De Cafézinho` | `xícara de cafézinho` |
| `g` | `g` |
| `kcal` | `kcal` |

## Parte 3 — Renderização condicional do bloco `apresentacao` (Fase 2)

### 3.1 Estrutura do novo bloco (quando disponível)

```typescript
interface ApresentacaoAlimento {
  nome_exibicao: string;
  detalhes?: string | null;
  modo_preparo?: string | null;
  fonte?: string;
  categoria?: string;
  confianca_parse: 'alta' | 'media' | 'baixa' | 'manual';
}
```

Este bloco é **opcional e aditivo**. Os campos atuais (`descricao`, `fonte_dados`, `grupo_alimentar.nome`) permanecem disponíveis.

---

### 3.2 Chain de prioridade para nome do alimento

```typescript
const getNomeExibicao = (alimento: Alimento): string => {
  // 1. Bloco apresentacao com confiança suficiente
  if (alimento.apresentacao?.nome_exibicao &&
      alimento.apresentacao.confianca_parse !== 'baixa') {
    return alimento.apresentacao.nome_exibicao;
  }

  // 2. Fallback formatado (Fase 1)
  if (alimento.descricao) {
    return formatFoodName(alimento.descricao);
  }

  // 3. Último recurso
  return '';
};
```

---

### 3.3 Regras de renderização por nível de confiança

#### `confianca_parse = 'alta'` ou `'manual'`

Renderização completa:

```
Linha 1: nome_exibicao                    → bold, cor primária
Linha 2: modo_preparo                     → regular, cor secundária (se existir)
Linha 3: categoria · fonte                → regular, cor terciária
          detalhes                         → tooltip no nome (se existir)
```

#### `confianca_parse = 'media'`

Renderização parcial — sem detalhes:

```
Linha 1: nome_exibicao                    → bold, cor primária
Linha 2: modo_preparo                     → regular, cor secundária (se existir)
Linha 3: categoria · fonte                → regular, cor terciária
```

Sem tooltip de detalhes.

#### `confianca_parse = 'baixa'`

Fallback seguro — usar campo `descricao` original, não `nome_exibicao`:

```
Linha 1: formatFoodName(descricao)        → bold, cor primária
Linha 2: categoria · fonte                → regular, cor terciária
```

Sem modo de preparo, sem detalhes. Exibir o nome formatado pela função utilitária.

### 3.5 Regra de supressão de redundância

Se `detalhes` for semanticamente redundante com `nome_exibicao`, ocultar o tooltip:

```typescript
const deveExibirDetalhes = (apresentacao: ApresentacaoAlimento): boolean => {
  if (!apresentacao.detalhes) return false;
  if (apresentacao.confianca_parse === 'media') return false;
  if (apresentacao.confianca_parse === 'baixa') return false;

  // Verificar redundância
  const nomeNorm = apresentacao.nome_exibicao.toLowerCase();
  const detalhesNorm = apresentacao.detalhes.toLowerCase();

  // Se todas as palavras de detalhes já estão no nome, é redundante
  const palavrasDetalhes = detalhesNorm.split(/[,\s]+/).filter(Boolean);
  const todasPresentes = palavrasDetalhes.every(p => nomeNorm.includes(p));

  return !todasPresentes;
};
```

---

### 3.6 Separador da sublabel

Categoria e fonte são separados por middle dot com espaço:

```typescript
const getSublabel = (apresentacao: ApresentacaoAlimento, alimento: Alimento): string => {
  const categoria = apresentacao?.categoria || alimento.grupo_alimentar?.nome || '';
  const fonte = apresentacao?.fonte || alimento.fonte_dados || '';

  if (categoria && fonte) return `${categoria} · ${fonte}`;
  if (categoria) return categoria;
  if (fonte) return fonte;
  return '';
};
```
