# Contrato Frontend — Tabela de Exercícios na Aba Performance

> **Para:** LLM Frontend Web (Painel de Longevidade)  
> **De:** Backend UNIO  
> **Data:** 15/03/2026  
> **Status:** ✅ Backend implementado e disponível em staging

---

## 1. Contexto

A aba **Performance & Funcionalidade** do Painel de Longevidade agora retorna o histórico de sessões de treino sincronizadas do Apple Watch via Apple Health. O frontend deve renderizar uma **tabela de exercícios** com filtros de intervalo temporal (7d, 30d, 90d), seguindo o padrão já existente nas demais abas do painel.

---

## 2. Endpoint

```
GET /api/painel_longevidade/clientes/{cliente_id}/performance-funcional?intervalo=7d
```

**Autenticação:** Bearer JWT (mesmo token já usado nas demais abas)

### Parâmetro de Query

| Param | Tipo | Valores | Padrão | Descrição |
|---|---|---|---|---|
| `intervalo` | `string` | `7d`, `30d`, `90d` | `7d` | Janela temporal do histórico de exercícios |

> [!IMPORTANT]  
> O parâmetro `intervalo` afeta **apenas** o bloco `historico_exercicios`.  
> Os demais biomarcadores (exercise_minutes, walking_speed, stability, etc.) continuam com suas janelas próprias (7d, 30d, 90d respectivas).

## 4. Schema dos Campos

### `historico_exercicios` (objeto)

| Campo | Tipo | Descrição |
|---|---|---|
| `intervalo` | `string` | Filtro aplicado: `"7d"`, `"30d"` ou `"90d"` |
| `total_sessoes` | `int` | Total de sessões no período |
| `resumo_por_tipo` | `object` | Agregação por tipo de exercício (ver abaixo) |
| `sessoes` | `array` | Lista de sessões individuais, ordenadas por data DESC |

### `resumo_por_tipo` (objeto dinâmico — chave = nome do tipo)

| Campo | Tipo | Descrição |
|---|---|---|
| `sessoes` | `int` | Quantidade de sessões desse tipo no período |
| `total_min` | `float` | Soma da duração em minutos |
| `total_kcal` | `float` | Soma das calorias ativas |

### Cada item em `sessoes[]`

| Campo | Tipo | Sempre presente? | Descrição |
|---|---|:---:|---|
| `id` | `int` | ✅ | ID interno da sessão |
| `tipo` | `string` | ✅ | Nome do exercício em pt-BR (ex: "Treino de Força") |
| `tipo_hk` | `int` | ✅ | rawValue do HKWorkoutActivityType |
| `inicio` | `string` | ✅ | Data e hora formato `"DD/MM HH:MM"` |
| `duracao_min` | `float \| null` | ✅ | Duração em minutos (1 casa decimal) |
| `calorias_kcal` | `float \| null` | ⚡ | Calorias ativas (pode ser null) |
| `distancia_metros` | `float \| null` | ⚡ | Distância em metros (null para musculação) |
| `elevacao_metros` | `float \| null` | ⚡ | Elevação ascendida (null para indoor/musculação) |
| `mets_medio` | `float \| null` | ⚡ | METs médio da sessão |
| `indoor` | `bool \| null` | ⚡ | True se exercício indoor |
| `fonte` | `string` | ✅ | Nome do dispositivo (ex: "PCTWatch") |

⚡ = disponível dependendo do tipo de exercício

## 5. Componentes de UI Esperados

### 5.1 Filtro de Intervalo (Segmented Control)

```
[ 7d ]  [ 30d ]  [ 90d ]
```

- Posicionado acima da tabela de exercícios
- Padrão selecionado: **7d**
- Ao trocar, refazer a chamada com `?intervalo=<valor>`
- Seguir o mesmo padrão visual dos filtros já existentes no painel

### 5.2 Resumo por Tipo (Cards ou Tags)

Renderizar o `resumo_por_tipo` como cards compactos acima da tabela:

```
┌─────────────────┐  ┌─────────────┐  ┌─────────────┐
│ 💪 Treino de     │  │ 🚴 Bicicleta │  │ 🚶 Caminhada │
│ Força            │  │              │  │              │
│ 2 sessões        │  │ 2 sessões    │  │ 1 sessão     │
│ 132 min          │  │ 7 min        │  │ 29 min       │
│ 643 kcal         │  │ 39 kcal      │  │ 201 kcal     │
└─────────────────┘  └─────────────┘  └─────────────┘
```

### 5.3 Tabela de Sessões

Colunas da tabela:

| Coluna | Campo fonte | Formatação | Visível mobile? |
|---|---|---|:---:|
| **Sessão** | `tipo` | Texto (nome pt-BR) | ✅ |
| **Início** | `inicio` | `DD/MM HH:MM` (já formatado) | ✅ |
| **Duração** | `duracao_min` | `X.X min` — se >= 60, formatar como `Xh Ymin` | ✅ |
| **Calorias** | `calorias_kcal` | `X.X kcal` — exibir `—` se null | ✅ |
| **Distância** | `distancia_metros` | Se > 1000: `X.X km`, senão `X m` — exibir `—` se null | ⚡ |
| **Elevação** | `elevacao_metros` | `X.X m` — exibir `—` se null | ⚡ |
| **METs** | `mets_medio` | `X.X` — exibir `—` se null | ⚡ |
| **Fonte** | `fonte` | Texto (ex: "PCTWatch") | ⚡ |

⚡ = pode ser ocultada em telas menores (responsivo)

### 5.4 Formatação de Duração

```javascript
function formatDuracao(min) {
  if (min == null) return '—';
  if (min >= 60) {
    const h = Math.floor(min / 60);
    const m = Math.round(min % 60);
    return `${h}h ${m}min`;
  }
  return `${min.toFixed(1)} min`;
}
```

### 5.5 Formatação de Distância

```javascript
function formatDistancia(metros) {
  if (metros == null) return '—';
  if (metros >= 1000) return `${(metros / 1000).toFixed(1)} km`;
  return `${Math.round(metros)} m`;
}
```

### 5.6 Estado Vazio

Se `total_sessoes === 0`:

```
Nenhuma sessão de treino registrada nos últimos {intervalo}.
Treinos do Apple Watch serão exibidos aqui automaticamente.
```

## 7. Referência Visual

A tabela deve seguir o mesmo design system já utilizado nas demais seções do Painel de Longevidade:

- **Dark mode** com tons de fundo `#1A1A2E` / `#16213E`
- **Linhas zebradas** com contraste sutil
- **Valores numéricos** alinhados à direita
- **Negrito** em sessões longas (> 30 min)
- **Dash (`—`)** para valores null, em cor muted

## 8. Chamada de API — Exemplo

```javascript
const intervalo = '7d'; // ou '30d' ou '90d'

const response = await fetch(
  `/api/painel_longevidade/clientes/${clienteId}/performance-funcional?intervalo=${intervalo}`,
  {
    headers: {
      'Authorization': `Bearer ${accessToken}`,
      'Content-Type': 'application/json',
    },
  }
);

const data = await response.json();
const { historico_exercicios } = data.biomarcadores;

console.log(historico_exercicios.total_sessoes);    // 5
console.log(historico_exercicios.resumo_por_tipo);  // { "Treino de Força": {...}, ... }
console.log(historico_exercicios.sessoes);           // [ {...}, {...}, ... ]
```
