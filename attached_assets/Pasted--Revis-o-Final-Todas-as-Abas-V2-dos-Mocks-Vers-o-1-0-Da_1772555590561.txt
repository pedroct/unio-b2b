## 6. Próximos Passos Recomendados

### Prioridade 1 — Resolver problema raiz de tokens

Criar uma diretriz explícita para a LLM do frontend:

```
REGRA: No Painel de Longevidade, elementos de identidade do módulo 
(valor do score, ícones do card, nomes de biomarcadores, linhas de 
gráfico, sparklines, bordas de cards ativos) NUNCA usam verde do 
sistema. Usam exclusivamente tokens --mod-longevidade-* baseados 
em Deep Indigo (#4A5899).

Verde do sistema (--sys-primary, #3B5F4A) é reservado para:
- Navegação principal da aplicação
- Avatar do usuário
- Headers/footers globais
```

Isso resolve os itens A1–A5 de uma vez.

### Prioridade 2 — Implementar inversão semântica da FCR

Criar lógica no componente `<BiomarkerCard />`:

```typescript
const INVERTED_METRICS = ['resting_hr']; // FCR: descendo = positivo

function getTrendColor(metric_type: string, direction: 'up' | 'down' | 'stable') {
  const isInverted = INVERTED_METRICS.includes(metric_type);
  if (direction === 'stable') return '--sys-text-muted';
  const isPositive = isInverted ? direction === 'down' : direction === 'up';
  return isPositive ? '--sys-success' : '--sys-warning';
}
```

### Prioridade 3 — Fechar decisões D1, D3, D6

Documentar no PRD como decisões tomadas. Sugestão de redação:

> **D1:** Avatar do paciente usa `--sys-primary` (verde institucional) em todas as abas. Avatar é identidade do usuário, não do módulo ativo.
>
> **D3:** Botão "Metas" removido da V1. Funcionalidade planejada para V2.
>
> **D6:** Títulos de empty states usam `--font-body` (Inter) weight 600. `--font-display` (Playfair Display) é reservado para valores numéricos de destaque (scores).
