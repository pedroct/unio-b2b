**Plataforma:** Web (Dashboard Profissional)

**Data:** 27/02/2026  
**Escopo:** Textos, copy, UX writing e ajustes visuais menores — não alterar lógica, chamadas de API ou estilos estruturais.

---

## Resumo da análise

A tela é funcional e bem organizada, com boa hierarquia visual entre refeições. No entanto, há **problemas recorrentes de capitalização nas unidades de medida**, inconsistências terminológicas e oportunidades de melhoria na clareza dos rótulos de macronutrientes. Foram identificados **18 ajustes**.

---

## Regra terminológica ativa: Paciente → Cliente

Usar **"Cliente"** em contextos administrativos e estruturais. Contextos clínicos (prescrição, plano alimentar) usam o nome da pessoa — já aplicado corretamente nesta tela ("Bruno Oliveira Santos").

### 10. ⚠️ Unidades de medida — Capitalização incorreta (problema recorrente)

Este é o problema mais crítico da tela. As unidades de medida estão com capitalização inconsistente e incorreta, usando Title Case quando deveriam usar sentence case (ou minúsculas quando são unidades de medida padrão).

**Regra:** Unidades de medida são substantivos comuns — devem estar em minúsculas ou sentence case, nunca Title Case.

| Atual | Novo |
|-------|------|
| `Fatia` | `fatia` |
| `Fatia Média (30g)` | `fatia média (30g)` |
| `Colher De Sopa` | `colher de sopa` |
| `Colher De Sobremesa` | `colher de sobremesa` |
| `Copo Grande` | `copo grande` |
| `Copo` | `copo` |
| `Concha Pequena Cheia` | `concha pequena cheia` |
| `Bifé` | `bifê` (verificar: possível erro de acentuação — a unidade correta seria "filé"? Ou é "bife" como porção? Se for unidade de medida "bife", manter `bife` em minúscula) |
| `Colher De Arroz/Servir` | `colher de arroz/servir` |
| `Copo Americano` | `copo americano` |
| `Fatia fina` | ✅ Já correto |
| `Grama` | `grama` |
| `Xícara De Cafézinho` | `xícara de cafézinho` |
| `Unidade` | `unidade` |
| `Copo De Requeijão` | `copo de requeijão` |

**Diagnóstico provável:** Os dados estão vindo do backend (possivelmente da tabela TACO ou Dietbox) com Title Case aplicado em cada palavra. A correção deve ser aplicada via formatação no frontend: converter todas as unidades para lowercase ao renderizar, ou aplicar `text-transform: lowercase` / transformação no componente.

**Implementação sugerida:**
```javascript
// Ao renderizar a coluna "Unidade"
const formatUnit = (unit) => unit.toLowerCase();
```

### 17. Separador decimal nos percentuais

**Problema recorrente:** Todos os percentuais do card de nutrientes usam **ponto** como separador decimal (`26.9%`, `47.8%`, `25.3%`).

**Correção:** Em pt-BR, o separador decimal é **vírgula**: `26,9%`, `47,8%`, `25,3%`.

**Implementação:**
```javascript
const formatPercent = (value) => value.toFixed(1).replace('.', ',') + '%';
```

## Regras gerais aplicadas (referência)

- **Terminologia: "Paciente" → "Cliente"** em contextos administrativos (nesta tela, usa-se o nome do cliente — correto)
- Nenhum texto de interface em ALL CAPS (inclui unidades: "KCAL" → "kcal")
- Títulos e subtítulos: sem ponto final
- Labels e links: sentence case
- Unidades de medida: sempre minúsculas (SI e uso geral)
- Separador decimal em pt-BR: vírgula, não ponto
- Nomes de alimentos: legibilidade humana, sem formato de tabela técnica
- Placeholders: sem reticências
- Mensagens de erro: `[O que aconteceu] + [O que fazer agora]`
- Tom: especialista amigável, não médico distante
